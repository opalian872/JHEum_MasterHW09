# JHEum_HW09

Unreal Engine C++ 멀티플레이어 숫자 야구 게임 과제입니다.

## 구현 과정

### 1. 멀티플레이 프로젝트 구성

- 강의 1-1~1-5를 참고하여 서버 테스트 환경과 `ChatMap`을 구성했습니다.
- GameMode, GameState, PlayerController, PlayerState, Pawn C++ 클래스와 Blueprint 클래스를 연결했습니다.

### 2. 멀티플레이 채팅 구현

- 강의 2-1~2-2를 참고했습니다.
- 클라이언트의 채팅 입력을 Server RPC로 전달하고, 서버가 Client RPC를 호출하여 모든 플레이어에게 출력하도록 구현했습니다.
- GameState의 NetMulticast RPC로 플레이어 접속 메시지를 출력했습니다.

### 3. 숫자 야구 기본 로직 구현

- 강의 3-1~3-3을 참고하여 서버 GameMode에 정답 생성과 S/B/OUT 판정 로직을 구현했습니다.
- 정답은 1~9 사이의 중복되지 않는 숫자 3개로 생성합니다.
- PlayerState에서 플레이어 이름과 현재/최대 시도 횟수를 관리하고 복제합니다.

### 4. 입력 검증 보완

- 과제 조건을 만족하도록 GameMode에서 정확히 3자리인지, 숫자로만 구성되었는지, 0 또는 중복 숫자가 포함되었는지 검사했습니다.
- 잘못된 입력은 안내 메시지만 출력하고 시도 횟수를 소진하지 않도록 했습니다.
- 유효한 숫자 입력과 시간 초과에 대해서만 시도 횟수를 1회 소진합니다.

### 5. 승리, 무승부, 리셋 구현

- 강의 3-4를 참고하여 `3S` 달성 시 승리 메시지를 표시하도록 구현했습니다.
- 모든 플레이어가 최대 시도 횟수 3회를 소진하면 무승부를 표시합니다.
- 리셋 시 새 정답을 생성하고 모든 플레이어의 시도 횟수를 초기화합니다.

### 6. 턴 및 타이머 도전 기능 구현

- 서버 GameMode가 현재 턴과 제한 시간을 관리하도록 구현했습니다.
- 현재 턴인 플레이어만 숫자를 입력할 수 있으며, 입력 후 다음 플레이어에게 턴을 넘깁니다.
- 시간이 종료되면 해당 플레이어의 시도 횟수를 1회 소진하고 다음 턴으로 이동합니다.
- 현재 턴, 남은 시간, 게임 진행 상태는 GameState의 Property Replication으로 동기화합니다.

### 7. 게임 시작 및 UI 구성

- 채팅 입력, 결과 공지, 타이머, 시작 버튼 Widget을 생성했습니다.
- 시작 버튼은 PlayerController의 Server RPC를 통해 서버 GameMode의 게임 시작 함수를 호출합니다.
- 승리 또는 무승부 후 데이터를 초기화하며, 시작 버튼으로 다음 게임을 진행할 수 있도록 구성했습니다.



## 실행 방법

1. PIE 플레이어 수를 2명 이상으로 설정합니다.
2. Dedicated Server 옵션으로 실행합니다.
3. 시작 버튼을 누른 뒤 현재 턴 플레이어가 채팅창에 숫자 3자리를 입력합니다.
